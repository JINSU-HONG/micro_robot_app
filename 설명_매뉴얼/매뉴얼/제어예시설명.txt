[제어 예시 2]

프로그램 실행 후 약 5초 대기
 - webcam 이 마이크로로봇의 최초 위치를 여러 번 센싱하여 평균 지점을 (0, 0) 으로 둔다.

b + enter 입력
 - 자기장, 힘 지령을 변경
m + enter 입력
 - 마우스 입력을 통한 위치 제어 (자기장 45도 고정, 힘 지령은 위치 제어기가 생성)

클릭 및 드래그
 - 마이크로로봇 위치 제어 목표 변경

0 입력 (화면 상에서는 0을 입력 안해도 됨)
 - 프로그램 종료 / 제어 중 바로 종료한 것으로 위험하다. init mode 로 변경(1 입력)한 뒤 0을 입력해 종료해야 했다.



[제어 예시 3]

프로그램 실행 후 약 5초 대기
 - webcam 이 마이크로로봇의 최초 위치를 여러 번 센싱하여 평균 지점을 (0, 0) 으로 둔다.

b + enter 입력
 - 자기장, 힘 지령 변경
0 + enter 입력
 - x축 정렬, 힘 0 (아직 init 모드 이므로 바로 정렬 되지 않는다.)

2 + enter 입력
 - run mode로 변경 (x축 정렬 됨)

b + enter 입력
 - 자기장, 힘 지령 변경
1 + enter 입력
 - y축 정렬, 힘 0 (run mode에 있으므로 바로 정렬 된다.)

1 + enter 입력
 - init mode로 변경

b + enter 입력
 - 자기장, 힘 지령 변경
7 + enter 입력
 - x축 정렬, x축 따라 sine 힘 지령 (바로 수행되지 않음)

2 + enter 입력
 - run mode로 변경 (x축 정렬, sine 진동)

w + enter 입력
 - 진동, 회전, 힘 등의 주파수를 변경
31.416 + enter  (영상에서는 6이 다음 줄로 넘어갔으나 콘솔 출력에 의한 것으로 한 번의 입력임)
 - 지령의 주파수를 5Hz 로 설정 (진동이 빨라짐)

1 + enter 입력
 - init mode로 변경

n + enter 입력
 - 제어하는 인버터 보드 (전류) 개수 변경
4 + enter 입력
 - 소형시스템이므로 4개의 인버터 보드 제어 (default도 4로 되어 있었음)

0 + enter 입력
 - 프로그램 종료 / 종료 전 init mode로 변경을 해야 한다. 지령 수행 중 갑자기 종료하면 전류 제어기는 1초간 더 지령을 수행한 후 종료된다.


[제어 예시 4] webcam을 통해 마이크로로봇을 정확히 인식하기 위한 threshold 변경 영상

HSV를 변경해가며 가장 마이크로로봇이 주변과 구분이 잘 되는 경우를 찾는다.
영상에서는 default로 둔 경우가 가장 잘 구분되는 경우이다.
밝기는 녹화될 영상의 밝기만 조정하며 마이크로로봇의 위치 인식에는 영향을 안 주도록 하였다.
